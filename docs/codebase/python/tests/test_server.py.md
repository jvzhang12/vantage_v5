# `tests/test_server.py`

This test file covers the HTTP surface of the app and the end-to-end flows built on top of it. It verifies health and workspace endpoints, opt-in HTTP Basic Auth around UI/API routes, memory and concept listing, concept and vault-note search, individual record lookups, chat response payload shape, canonical product-safe `recall` payloads plus the legacy `working_memory` alias, response-mode payload fields, safe top-level `system_state` and `activity` payloads, top-level `turn_interpretation` payloads for normal chat and Scenario Lab turns, Scenario Lab chat answers including the recommendation/tradeoff brief rather than only artifact-created copy, Navigator `control_panel` exposure, `apply_protocol` actions that load the built-in Scenario Lab protocol into chat and Scenario Lab working memory, top-level `semantic_frame` payloads that expose the user goal/task/surface/follow-up contract, top-level `semantic_policy` payloads that can clarify or locally handle save/publish/experiment commands before generic chat, plain `why` questions staying in direct chat instead of being treated as context-inspection requests, protocol API listing/fetch/update including built-ins and experiment-scoped edits, protocol learning where explicit email preferences upsert a stable `email-drafting-protocol`, `/api/protocols` serialization, protocol recall for later email drafts, in-place protocol updates for changed signatures, protocol records being rejected as whiteboard-open targets, tolerant parsing of same-line whiteboard offer tags, verifier-guided structured normalization that can turn natural-language model output into a pending whiteboard offer, provider-failure fallback that stays product-safe, fallback whiteboard drafts for accepted offers or fresh work products, sanitized Scenario Lab failure messages, and narrow draft-constraint guards for no-em-dash, short optional-reading, and known-signature placeholder repair requests.

That matters because these routes are the contract for the whole application. The tests ensure the server exposes consistent data, routes requests to the right storage scope, avoids writing when it should only draft, keeps experiment data separate from durable repository state, exercises the navigator-first Scenario Lab path, confirms both conservative and interpreter-driven continuity anchoring, verifies grounded turns expose the exact grounded context surfaced for the final answer rather than only recalled working-memory hits, ensures best-guess turns are disclosed honestly in the visible assistant reply, locks in the newer behavior where the interpreter’s chosen path is visible through a compact `turn_interpretation` payload, verifies that the adjacent `semantic_frame` exposes product-level understanding without replacing route policy, verifies that `semantic_policy` can save/publish visible whiteboard content locally, clarify ambiguous save requests, and answer experiment status locally without calling generic chat, confirms that concrete work products can first invite whiteboard collaboration and then return pending whiteboard proposals without being silently applied or promoted into durable memory, verifies that pending whiteboard context is reported only when the server had the original prior-turn prompt and a client-matched carry phrase to justify it, verifies the C3 alias bridge where pending whiteboard `type` can still recover the canonical `status`, verifies that stale pending offers are dropped when a hidden whiteboard should stay out of scope, verifies that even a supplied live whiteboard buffer stays out of model scope when The multi-user profile coverage verifies that a configured Basic Auth user map creates isolated Markdown storage per username, rejects cross-user credentials, keeps `/api/health` safe for unauthenticated uptime checks, prevents one user's draft/artifact trail from appearing in another user's workspace or memory API, supports local account creation with hashed password storage and seeded per-user folders, and keeps user-entered OpenAI API keys scoped per profile, masked in responses, memory-only, and able to override then clear back to the environment key.
