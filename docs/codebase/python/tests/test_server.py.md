blocking, secure cookie configuration, read-only calendar day payloads, memory and concept listing, concept and vault-note search, individual record lookups, chat response payload shape, canonical product-safe `recall` payloads plus the legacy `working_memory` alias, response-mode payload fields, safe top-level `system_state` and `activity` payloads, top-level `turn_interpretation` and `surface_invocation` payloads for normal chat and Scenario Lab turns, Scenario Lab chat answers including the recommendation/tradeoff brief rather than only artifact-created copy, Navigator `control_panel` exposure, `apply_protocol` actions that load the built-in Scenario Lab protocol into chat and Scenario Lab working memory, top-level `semantic_frame` payloads that expose the user goal/task/surface/follow-up contract, top-level `semantic_policy` payloads that can clarify or locally handle save/publish/experiment commands before generic chat, plain `why` questions staying in direct chat instead of being treated as context-inspection requests, protocol API listing/fetch/update including built-ins and experiment-scoped edits, protocol learning where explicit email preferences upsert a stable `email-drafting-protocol`, `/api/protocols` serialization, protocol recall for later email drafts, in-place protocol updates for changed signatures, protocol records being The fixture also creates clean test `state/` and `traces/` directories so local manual experiments or trace files cannot leak into server assertions.

The visible-artifact chat regression coverage verifies that ordinary follow-up Q&A over an open Whiteboard, such as asking what to do first from the Midterm Study Plan, asking to summarize/explain/key-point that plan, or using pronoun-only references like "summarize it", stays chat-first, keeps the visible artifact in the payload, and does not save a derivative artifact, draft, graph action, or concept.

The attention/surface regression coverage verifies that saved-material lookup can produce a Navigator/control-panel Whiteboard open directive even when an already-visible Today card is also selected, and that the open directive remains separate from draft/save behavior through the full server response path. It also verifies that same-title opened Whiteboard copies do not displace their selected source artifact as the primary open target for lookup/open turns, and that an explicit final `open_only` Whiteboard invocation forces chat execution when an earlier base route was drafty, preventing workspace updates, automatic graph writes, graph actions, artifact actions, operational surfaces, or created records for a UI-only open.

The trace observability coverage verifies that JSON turn traces retain their existing low-level fields while also appending a `final_response` section that matches the browser-facing `/api/chat` response after server post-processing, including Today surface payloads, saved-artifact open-only directives, selected attention resources, final no-write fields for ordinary chat, and a request history snapshot capped to the existing last-six trace limit.

The fixture also ignores generated `launch-strategy*.md` workspace files when copying local workspaces into the temporary repo, so manual Scenario Lab runs cannot change detached branch namespace assertions.

The optional Basic Auth coverage also asserts that authenticated users receive the production-facing `Vantage` shell brand rather than the earlier development-version masthead.

The PWA coverage verifies that root install assets expose the expected manifest/service-worker/icon headers, keep service-worker scope at `/`, and stay public before login while protected API routes remain locked behind auth.

The calendar endpoint coverage verifies that `/api/calendar/day` is protected in profile-auth mode, rejects unsupported date phrases with a clear 400, and returns the read-only day payload shape that the calendar artifact surface will consume.

The calendar/task profile coverage verifies that, when no explicit global JSON provider path is configured, authenticated profiles read operational demo data from their own `users/<profile>/state/calendar/events.json` and `users/<profile>/state/tasks/tasks.json` files instead of leaking another profile's schedule or task list.

The surface-invocation coverage verifies that durable work products such as email drafts now auto-draft into the whiteboard even when the interpreter would have kept them in chat or offered first, while explicit composer `offer` still preserves the old invitation behavior for ambiguous/manual flows.

The multi-user profile coverage verifies that a configured Basic Auth user map creates isolated Markdown storage per username, rejects cross-user credentials, keeps `/api/health` safe for unauthenticated uptime checks, prevents one user's draft/artifact trail from appearing in another user's workspace or memory API, supports local account creation with hashed password storage, seeded per-user folders, and empty profile-local calendar initialization when no global calendar file is configured, can require an access code before public account creation, exposes canonical default protocols without copying them into user folders, persists user edits as private overrides, keeps a profile literally named `canonical` from making private records look canonical in list/detail/protocol payloads, and keeps user-entered OpenAI API keys scoped per profile, masked in responses, memory-only, and able to override then clear back to the environment key.

The canonical-scope regression coverage verifies that a profile literally named `canonical` keeps private concept, protocol, memory, and artifact records labeled durable while configured canonical defaults remain read-through and canonical.

The saved-item open coverage verifies that `/api/concepts/open` returns additive source/copy provenance, including canonical source scope for read-through records and durable writable scope for the opened user Whiteboard copy.

The chat-service monkeypatch stubs now expose `ChatTurnBodyParts` through `to_body_parts()`, matching the deep `turn_payloads.py` assembler boundary instead of returning prebuilt public turn dictionaries.
