# Compatibility Ledger

> Status: Current source of truth
> Note: This ledger documents compatibility surfaces that are intentionally retained. It is not removal authorization, and it does not change public payloads, storage paths, routing, retrieval, generation, TurnPlan authority, write execution, tests, or frontend behavior.

Date: 2026-05-21

## Purpose

Vantage carries a few older names and payload aliases while the product language, React client, and backend authority model continue to settle. These surfaces are easy to mistake for desired future product shape. Treat this ledger as the first stop before renaming, deleting, or tightening compatibility behavior.

## Rules For Future Cleanup

- Do not remove a compatibility field just because current React appears not to need it; check tests, smokes, and any external client assumptions.
- Do not rename storage paths as part of product-language cleanup.
- Prefer adding status labels and removal conditions before deleting aliases.
- If a compatibility surface affects behavior, label the relevant tests as compatibility, fallback/safety, or current product contract before changing them.
- Public `/api/chat` shape changes require backend, frontend, tests, docs, and smoke coverage in the same planned migration.
